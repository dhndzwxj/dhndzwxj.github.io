1. 在VMware中安装群晖7.2

1.0 前期下载

①StarWind V2V Image Converter 镜像转换工具：https://www.starwindsoftware.com/tmplink/starwindconverter.exe

②ARPL编译系统

  • 汉化版Github Release 页面：https://github.com/wjz304/arpl-zh_CN/releases
  • 英文版Github Release 页面：https://github.com/fbelavenuto/arpl/releases

下载DSM操作系统（也可以不下载，本地安装速度会快一点）【推荐DS918_7.2-64570.pat】

1.1 制造引导

①将下载下来的 arpl-xxxxxxxx.img.zip 解压出来，文件为 arpl.img

②打开 StarWind V2V Image Converter 软件，选择 Local file

③选择刚刚解压出来 arpl.img

④选择 Local file

⑤选择 VMDK

⑥选择 VMware Worksatation pre-allocated image

⑦选择输出路径 > Convert

⑧转换成功 > Finish

这个时候，你会得到两个文件，分别为： arpl.vmdk 和 arpl-flat.vmdk 。

相关文件也可以在这个网盘中下载：https://pan.chenlungang.com/阿里云盘/系统/黑群晖

1.2 配置虚拟机

①打开VMware【选择 Workstation 16.x 或者 15 ，兼容性稍微好一点，不用选择最新的】，新建虚拟机。然后选择“稍后安装操作系统”。

②选择 Linux ，版本按照图上设置就行

③设置虚拟机名称和位置。

④配置处理器，2核其实就够了。

⑤设置内存，亲测4G绝对够用！

⑥网络类型，这边要选择使用桥接网络。【后文有桥接网络的设置】

⑦I/O控制器，默认。

⑧磁盘类型，选择 SATA，选择 使用现有虚拟磁盘【这时候需要自己搞一个VMDK文件】。

选择 保持现有格式。接下来可以进一步添加一个磁盘，这回选择“创建新虚拟磁盘”，自己分配大小就可以。

1.3 制作并安装引导

接下来启动虚拟机。直接就能安装成功了。

3. 在VMStation上设置网络桥接模式

3.1 VMware虚拟机三种联网方法及原理

  • Brigde桥接：
    • 默认使用VMnet0。Bridge 桥”就是一个主机，这个机器拥有两块网卡，分别处于两个局域网中，同时在”桥”上，运行着程序，让局域网A中的所有数据包原封不动的流入B，反之亦然。这样，局域网A和B就无缝的在链路层连接起来了，在桥接时，VMWare网卡和物理网卡应该处于同一IP网段，当然要保证两个局域网没有冲突的IP。
    • VMWare 的桥也是同样的道理，只不过，本来作为硬件的一块网卡，现在由VMWare软件虚拟了！当采用桥接时，VMWare会虚拟一块网卡和真正的物理网卡就行桥接，这样，发到物理网卡的所有数据包就到了VMWare虚拟机，而由VMWare发出的数据包也会通过桥从物理网卡的那端发出。所以，如果物理网卡可以上网，那么桥接的软网卡也没有问题了，这就是桥接上网的原理了。 这一种联网方式最简单，在局域网内，你的主机是怎么联网的，你在虚拟机里就怎么连网。把虚拟机看成局域网内的另一台电脑就行了！
  • NAT网络地址转换 ：默认使用VMnet8
    • NAT 是 Network address translate的简称。NAT技术应用在internet网关和路由器上，
      比如192.168.0.123这个地址要访问internet，它的数据包就要通过一个网关或者路由器，
      而网关或者路由器拥有一个能访问internet的ip地址，这样的网关和路由器就要在收发数据包时，对数据包的IP协议层数据进行更改（即 NAT），以使私有网段的主机能够顺利访问internet。此技术解决了IP地址稀缺的问题。同样的私有IP可以网关NAT 上网。
    • VMWare的NAT上网也是同样的道理，它在主机和虚拟机之间用软件伪造出一块网卡，
      这块网卡和虚拟机的ip处于一个地址段。同时，在这块网卡和主机的网络接口之间进行NAT。虚拟机发出的每一块数据包都会经过虚拟网卡，然后NAT，然后由主机的接口发出。
    • 联网方式：动态IP、静态IP

bridge

3.2 设置虚拟机的桥接网络

在VMware中，选择“设置——虚拟网络编辑器”。

点击更改设置。然后点击“添加网络”，名称选择VMet0，VMet信息选择桥接模式，桥接到自己的网卡。

下一步设置虚拟机，设置为桥接模式。

这样就能在路由器上搜到自己的群晖设备了！

  • 首选DNS服务器：114.114.114.114
  • 备用DNS服务器：114.114.115.115

4. WebStation做静态网站托管

在套件中心中下载WebStation套件。相应地下载PHP、Git、Nginx等套件。

打开该套件，点击“网页服务”，名称、描述可以自己定义，选好文档根目录【可以在File Station下的web文件夹下自己新建一个文件夹，建议取名为public】。HTTP后端服务器可以选择Nginx。

然后在“网络门户”中设置服务。主要是编辑端口。【可惜80和5000等端口已被占用，设置个其他的端口即可，HTTP和HTTPS的端口不能设置成一样的】

然后把个人网站的相关文件上传至`文档根目录public`！

最后在浏览器中输入http://[主机名]:218或者https://[主机名]:729即可查看自己的网站了。

4a. 通过scp命令上传文件到NAS

如果上一节中的把个人网站的相关文件上传至文档根目录public！这一步，可以用代码实现，就非常方便了！

尝试：把dlog下的public文件夹上传到nas的代码：

scp -r local_dir username@servername:remote_dir
scp -r public dhndzwxj@192.168.3.150:/volume1/web/
scp -r -O public dhndzwxj@192.168.3.150:/volume1/web/

也可以上传单个文件到nas

scp /path/filename username@servername:/path/

例如scp /var/www/test.php root@192.168.0.101:/var/www/ 把本机/var/www/目录下的test.php文件上传到192.168.0.101这台服务器上的/var/www/目录中。

也可以从nas上下载整个目录

scp -r username@servername:/var/www/remote_dir/[远程目录] /var/www/local_dir[本地目录]

也可从nas上下载单个文件

scp username@servername:/path/filename /var/www/local_dir[本地目录]

例如scp root@192.168.0.101:/var/www/test.txt把192.168.0.101上的/var/www/test.txt的文件下载到/var/www/local_dir[本地目录]

5. 群晖NAS设置外网访问

完整步骤：公网IP+DNSPOD域名解析+光猫桥接+路由器端口转发+DDNS解析+WebDAV

5.1 申请域名

阿里云、腾讯云都可以买

5.2 申请公网IP

打电话给电信10000，转人工服务，就说要申请公网IP，就会申请。

如果需要理由，就说摄像头远程监控家里情况等。

（移动的网络，情况不一样，后续再补充）

获取公网IP地址：<http://members.3322.org/dyndns/getip >

5.3 在DNSPOD添加域名和记录（域名解析）

先完成实名验证，才会开始解析:<https://dnspod.cn >

域名解析就是把 域名 和IP地址对应起来。

5.4 光猫设置桥接

从外网访问群晖主机：https://www.jianshu.com/p/ee61b0b58854

5a. Zerotier实现外网访问

先在套件中心下载“Zerotier”。

然后在Zerotier官网申请一个账户。然后在主页创建一个新的网络【Create A Network】，系统会自动生成一个16位的虚拟网络ID。

接下来选择一个喜欢的网段。

复制16位的虚拟网络ID，然后进入群晖的root后台

ssh dhndzwxj@192.168.3.150
sudo -i
sed -i 's/package/root/g' /var/packages/zerotier/conf/privilege
cd /var/packages/zerotier/target/bin
./zerotier-one -q join fada62b01504eb9c [16位的虚拟网络ID]
[同时在回车前，打开nas的zerotier点击启动再回车。]
[显示“200 join OK”即表示连接成功.]

再回到Zerotier页面，可以发现一个新申请加入的用户，可以勾选就同意其加入网络。可以进行命名方便管理。

新设备加入网络后，会自动给它分配一个IP。通过这个IP，我们就可以访问NAS了。

在其他设备上安装Zerotier也同理，只不过记得在官网上勾选允许新用户加入网络。
